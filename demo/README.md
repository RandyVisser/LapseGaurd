# Demo kit — condo.insure walk-through

Everything you need to record the two-sided product demo (manager + unit owner).

## What's in here

| File | Use |
|------|-----|
| `walkthrough-script.md` | Shot-by-shot script: scenes, exact clicks, narration, timing. |
| `sample-units.csv` | The owner list the manager imports (9 units, Florida condo). |
| `sample-dec-page.pdf` | Maria Garcia's HO-6 declaration page — built to **pass** the demo requirements. |

> The data ties together on purpose: the CSV's **Unit 204 = Maria Garcia** at
> **1420 Seabreeze Blvd Apt 204**, and the dec page is issued to *Maria Garcia* at that same
> address with coverage that clears the requirements set in Scene 1. That's what makes the
> demo end on a green "You're covered."

## Recording setup tips

**Capture two sides clearly.**
- Use **two browser windows/profiles** (manager vs. owner), or one normal + one incognito.
- A subtle on-screen label ("Manager view" / "Owner view") or a quick title card between acts
  makes the hand-off obvious. A lower-third caption works too.

**Show the invite email arriving.**
- Easiest: set Unit 204's email to a Gmail **`you+demo204@gmail.com`** alias — it lands in your
  normal inbox but reads as a distinct owner address.
- Don't want to show a real inbox? Preview the exact email from the manager side first
  (**Settings → Email Previews → Invite**), then cut to the real click-through. The invite link
  only exists in the delivered email, so you still need to receive one to open `/join/...`.

**Keep it legible & clean.**
- Browser zoom **110–125%**; hide bookmarks bar, extensions, and personal tabs.
- Record at **1080p**; if you'll post on mobile/social, also frame for a 1:1 or 9:16 crop.
- Tools: **Loom** or **macOS Screenshot (⇧⌘5)** for a quick take; **ScreenStudio** or **OBS**
  for auto-zoom/polish; **CleanShot X** for a tidy cursor + click highlights.

**Protect real data.**
- Record against a **throwaway association** you create just for the demo (the script does this).
- Use only the seeded demo names/emails here — never a real owner's PII.
- When done, delete the demo association: **Settings → Danger Zone → Delete Association**.

**Pace the AI moment.**
- After the owner clicks **Submit**, the page shows "Reading your document…" for ~10–30s and
  updates itself. Don't cut early — let it resolve to the green hero so the payoff is on camera.
  If you want a tighter edit, speed-ramp that waiting beat 2–4× in post.

## Quick start

1. Open `walkthrough-script.md` and skim the **Pre-production checklist**.
2. Edit `sample-units.csv` → set **Unit 204's** email to a real inbox you control.
3. Record Act 1 (manager) then Act 2 (owner), following the scenes.
4. Delete the demo association afterward.
